# Security Policy

## Supported Versions

The current release line is experimental but maintained:

- `0.1.x`

Older versions may not receive fixes.

## Reporting a Vulnerability

Please do not post undisclosed security issues in public issues.

Use GitHub's private vulnerability reporting flow for this repository when possible:

- Open the repository on GitHub
- Go to the Security tab
- Use "Report a vulnerability"

If private reporting is not available in your environment, open an issue only for non-sensitive discussion and avoid posting exploit details.

## Scope

This project is a developer tool. Security reports are most useful when they involve:

- command execution behavior
- unsafe file mutation behavior
- credential handling in backend integrations
- malicious prompt or output handling that can cause unintended local effects
